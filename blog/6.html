Authority


In this blog, we’re diving into the world of ports—and no, not the boat kind. We’re talking about the numbered network ports you’ll need to recognize for the A+ certification exam. There are quite a few, so I’ll try to cover as many as I can without making your eyes glaze over.

We’ll be going in numerical order, because that just makes life easier. Let’s get started!

Port 21 – FTP (File Transfer Protocol)

FTP is short for File Transfer Protocol, and as the name suggests, it’s used for transferring files. This port is commonly associated with programs like FileZilla, which let you move files between a local machine and a remote device.

One common use case: transferring files to an IoT device like a Raspberry Pi. It beats pulling out the SD card every time. It’s worth noting, though, that most modern web browsers no longer support FTP natively (as of 2021), so you'll need third-party software for this task.

Port 22 – SSH (Secure Shell)

SSH stands for Secure Shell (not “SSP,” I know, weird). It allows you to securely connect to another computer—typically a server—and run commands remotely. If you’ve ever used a Raspberry Pi without a monitor and keyboard, SSH is your best friend.

It’s a secure and efficient way to access remote systems without needing a full graphical interface.

Port 23 – Telnet

Next up is Telnet, which operates on port 23. Now, fun fact: Telnet stands for Teletype Network—not “Telecom Network” like you might expect. Telnet is also used to remotely access computers, similar to SSH.

The downside? It’s not secure. Data sent via Telnet is in plain text, so it's mostly been replaced by more secure alternatives like SSH. It still pops up in legacy systems, though, so it's worth knowing.

Port 25 – SMTP (Simple Mail Transfer Protocol)

SMTP is the protocol used to send emails. Whether you think of it as a “simple protocol to transfer mail” or a “protocol to transfer simple mail,” the end result is the same: it helps your messages get from point A to point B.

Most people don’t interact with SMTP directly, since third-party email providers like Gmail or Yahoo handle all the backend work. But if you're running your own email server, configuring SMTP is a must. (POP3 and IMAP also come into play here—but we’ll save those for another day.)

Port 53 – DNS (Domain Name System)

Let’s finish this post with a heavy hitter: DNS, which runs on port 53. This one might be the most important protocol of all, because without it, you wouldn’t be able to type in google.com and end up at Google’s homepage.

DNS works by taking a domain name and translating it into an IP address. The cool part? That IP address might be different depending on your location, since companies like Google have data centers all over the world. So DNS helps route you to the fastest or nearest server, making the experience smoother for everyone.

Wrapping Up

We’ve just scratched the surface of the many ports you’ll need to remember for the A+ exam. From file transfers and remote access to sending email and navigating the web, these protocols make up the foundation of modern network communication.

We’ll cover more ports in a future blog, but for now, you’ve got a solid start on the essentials. Memorize those port numbers—they’re guaranteed to show up on the test (and in real-world scenarios too).
